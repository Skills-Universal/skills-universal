# Frontend UI Engineering

## Overview

Build production-quality user interfaces that are accessible, performant, and visually polished. UI that looks like it was built by a design-aware engineer — not generated by an AI.

## When to Use

- Building new UI components or pages
- Modifying existing user-facing interfaces
- Implementing responsive layouts
- Fixing visual or UX issues

## Avoid the AI Aesthetic

| AI Default | Problem | Production Quality |
|---|---|---|
| Purple/indigo everything | Models default to "safe" palettes | Use the project's actual color palette |
| Excessive gradients | Visual noise | Flat or subtle, matching design system |
| Rounded everything | Ignores hierarchy of corner radii | Consistent border-radius from design system |
| Lorem ipsum copy | Hides layout problems | Realistic placeholder content |
| Oversized padding everywhere | Destroys visual hierarchy | Consistent spacing scale |
| Shadow-heavy design | Competes with content | Subtle or no shadows |

## Component Architecture

- Prefer composition over configuration
- Keep components focused (one thing)
- Separate data fetching from presentation
- Avoid prop drilling deeper than 3 levels

## State Management — Choose Simplest

```
local useState       → component-specific UI state
lifted state         → shared between 2-3 siblings
Context              → theme, auth, locale (read-heavy)
URL searchParams     → filters, pagination, shareable state
Server state         → remote data with caching
Global store         → complex client state app-wide
```

## Accessibility (WCAG 2.1 AA)

- Every interactive element keyboard accessible
- ARIA labels on elements without visible text
- Focus management when content changes
- Meaningful empty, loading, and error states
- Never use color as sole indicator of state

## Responsive Design

- Mobile first, then expand
- Test at: 320px, 768px, 1024px, 1440px

## Red Flags

- Components > 200 lines
- Inline styles or arbitrary pixel values
- Missing error/loading/empty states
- No keyboard navigation
- Generic AI look (purple gradients, oversized cards)

## Verification Checklist

- [ ] Renders without console errors
- [ ] All interactive elements keyboard accessible
- [ ] Works at 320px, 768px, 1024px, 1440px
- [ ] Loading, error, empty states all handled
- [ ] Follows project's design system
- [ ] No accessibility warnings
